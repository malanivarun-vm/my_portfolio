Products I've built
VitaeHealthcare / Family records

A family health record system where AI helps structure the past, not just read one document

Vitae helps families upload prescriptions and lab reports, review AI-structured outputs, store them as a long-term health record, and build context across older prescriptions over time. The product challenge is not just OCR accuracy — it is designing a trustworthy repository where extraction, review, storage, and later analysis all work together in a sensitive healthcare workflow.

AI classification + extraction Confidence-aware review Long-term document repository Historical prescription context
AndiamoConsumer / Group coordination

A coordination layer for the part of trip planning nobody really owns

Andiamo is built around the insight that group trips usually fail before booking starts. The real issue is not itinerary planning — it is coordination, commitment, budget alignment, and decision-making across a group. The product uses deadlines, private budgets, and lightweight social pressure to move a group toward a decision.

Commitment-first flow Budget privacy by design WhatsApp-compatible sharing
RemiFounder tools / Sales memory

A founder CRM built around the idea that remembering context matters more than filling a pipeline

Remi is a sales memory system for early-stage founders who do not behave like trained sales teams. It captures post-call context through lightweight inputs, uses AI to structure what mattered, and brings that context back when follow-up time comes. The core thesis: do not ask founders to become better at CRM admin — build a system that adapts to how they already work.

Telegram bot capture flow AI extraction + confirmation Context-aware follow-up system
LumiAssessment / Growth tools

A personality and skill assessment app — and a lesson in where systems complexity hides

Lumi is a personality and skill assessment app with Google login, personalised results, and a dashboard. The visible product is clean. The harder part was making authentication work reliably across server-rendered flows without silent failure — a class of bug where the system looks healthy until the next request proves otherwise.

Google OAuth Next.js + Supabase Auth Server-rendered session handling

BlinkitQuick commerce / Dark store ops

A mobile command layer for Blinkit's dark store managers — built around the insight that peak-hour failure is an information problem

DSM Operational Hub is a case study and live prototype for a real-time decision support tool for Blinkit dark store managers. Primary research identified that during peak hours, the one person with decision authority on the floor has the least real-time information. The product closes that gap — without adding cognitive load to an already chaotic shift.
