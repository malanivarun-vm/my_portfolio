Remi · founder CRM

A sales memory system for founders who do not have time to behave like a sales team

Remi is built for early-stage founders who run sales themselves and lose deal context because every existing CRM assumes someone will do the admin later. The product captures conversation context with minimal effort, uses AI to structure what matters, and surfaces that context when follow-up time comes.

Prototype live Telegram bot Claude API Supabase
Product thesis

The problem is not weak CRM features. It is a wrong mental model.

Research with founders and CRM communities pointed to the same failure mode: people do not abandon CRMs because pipelines are missing fields. They abandon them because logging and maintaining context feels like extra work after the real work is already done. Remi is built around that insight.

What I built

A lightweight capture flow that fits existing founder behaviour

  • Telegram bot capture flow for text, forwarded content, and quick post-call inputs.
  • AI extraction of contact, company, summary, deal signals, and next action.
  • Confirmation step before saving, so trust and control stay explicit.
  • Dashboard with pipeline view, context lookup, and linked Telegram authentication.
Why it matters

Every CRM was built for salespeople. Founders are not salespeople.

Research with founders and CRM communities pointed to the same failure: 73% of leads are never followed up — not from laziness, but because founders lose the context behind each conversation. What was the problem they mentioned? What did they hint at on budget? What was actually agreed as the next step? That context lives only in the founder's head, and it fades fast. Remi is built around that finding: the failure is not weak pipeline features, it is tools asking founders to change how they work. Remi adapts to existing behaviour instead.
