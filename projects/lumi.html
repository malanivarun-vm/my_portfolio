Lumi · assessment app

A good reminder that product simplicity often hides systems complexity

Lumi is a personality and skill assessment app with Google login, results access, and a personalised dashboard. The visible product is clean. The harder part was making authentication actually work across server-rendered flows without silent failure.

In development Google OAuth Next.js Supabase Auth
What made it hard

The dangerous bugs were the ones that looked like success

A helper built for one runtime context silently swallowed the cookie write needed for OAuth callback handling. The system looked healthy until the dashboard failed to recognize the session on the next request.

Key lessons

This is a build about judgment, not just debugging

  • Always ask which runtime context a helper was actually designed for.
  • Missing values and wrong values are different failure modes; fallback logic only covers one of them.
  • When a bug chain gets long, the visible error is usually downstream from the real cause.
Portfolio value

Useful as a builder story, weaker than Vitae or Andiamo as a flagship

Lumi works best in the portfolio as evidence of technical judgment and debugging depth. It is less compelling as a core product story than the other two projects, but it adds texture to how I learn and build.
