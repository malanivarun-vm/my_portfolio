Vitae · healthcare PWA

Helping families make sense of medical documents with AI

Vitae is a PWA where users upload prescriptions or lab reports, AI structures the contents, and the product asks the user to review before anything becomes permanent. The product challenge is not just extraction accuracy. It is designing trust into every handoff.

Team build · PWA Next.js OpenRouter / Gemini Supabase
What I built

The upload, processing, and review flow

I owned the section where a user picks a document, the system classifies it, extracts the structure, exposes low-confidence fields, and only then moves toward saving. That meant solving both UI clarity and AI pipeline reliability together.

Why it matters

AI is useful here only if it stays honest about what it does not know

The product insight behind Vitae is not the extraction itself. Medical documents require trust in two directions: the user needs to trust that the AI read the document correctly, and the user needs to trust that their data is stored safely. The confidence-aware review UI handles the first — low-confidence fields are surfaced explicitly so the user knows what to verify. LocalStorage before auth handles the second — the user sees extracted value before being asked to create an account. Every architectural decision in the upload flow was made in service of making trust explicit rather than assumed.

What it taught me

Trust design is the real product layer

The most interesting decisions were not model choices. They were about when to trust AI, when to interrupt the user, what data should be editable, and how to avoid implying certainty where the user cannot actually verify a field from memory.
